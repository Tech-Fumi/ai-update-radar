# Experiments - PoC置き場

Layer 3 に分類された項目の実験を行う隔離環境。

## ルール

### 時間制限

- **上限: 30〜90分**
- 超えたら強制終了
- 「もう少しで動きそう」は罠

### 隔離原則

- 本体リポジトリには入れない
- 依存は最小限
- 失敗しても影響なし

### 記録必須

すべての実験に以下を記録：

```markdown
# Experiment: {名前}

## 概要
- 日付:
- 対象:
- 仮説:

## 結果
- 所要時間:
- 結論: 採用 / 不採用 / 継続検討

## 学び
- 良かった点:
- 問題点:

## 次のアクション
- 採用の場合: どのリポジトリに入れるか
- 不採用の場合: なぜダメだったか
```

## ディレクトリ構成

```
experiments/
├── README.md
├── YYYY-MM-DD-{名前}/
│   ├── README.md      # 実験記録（必須）
│   ├── src/           # 実験コード
│   ├── data/          # テストデータ（.gitignore対象）
│   └── results/       # 結果（.gitignore対象）
└── _templates/
    └── experiment.md  # テンプレート
```

## 判断基準

### 採用条件

以下を**すべて**満たす場合のみ採用：

1. 90分以内に動作確認できた
2. 既存システムに破壊的変更なく統合できる
3. コスト（時間 or 費用）の削減が見込める
4. 運用負荷が増えない

### 不採用でも価値がある

- 「なぜダメだったか」の記録
- 将来の再検討時の参考
- 同じ失敗を繰り返さない
