# AI Update Radar 開発ルール

## プロジェクト概要

AIの本質的な更新（能力・制限・価格）を検知・評価・実験するための運用リポジトリ。

**立ち位置**:
- ❌ 新しいAIツール集
- ❌ ニュースまとめ
- ✅ AI進化を"使える変化"に変換するフィルター

---

## 構成

```
ai-update-radar/
├── sources/           # 監視対象定義
├── collectors/        # 収集スクリプト
├── evaluators/        # 影響判定ロジック
├── experiments/       # PoC置き場（隔離）
├── docs/weekly/       # 週次サマリ
├── exports/           # 他リポジトリへの成果物
└── .private/          # 非公開データ（.gitignore対象）
```

---

## 運用ルール

### 1. 情報の3層分類

| Layer | 対象 | アクション |
|:-----:|------|-----------|
| 1 | 90%のノイズ | 無視（見ない） |
| 2 | 変化検知のみ | 記録・週次サマリ |
| 3 | 深掘り対象 | 30-90分で実験 |

### 2. 検知すべき変化（Layer 2 以上）

- **能力変化**: 新モデル、新機能、Agent拡張
- **制限解除**: コンテキスト長、レート制限緩和
- **価格変化**: 料金改定、新プラン

### 3. 実験ルール（Layer 3）

- **時間上限**: 30〜90分
- **隔離環境**: experiments/ 内のみ
- **記録必須**: README.md で結果を記録
- **採用基準**: 4条件すべて満たす場合のみ

### 4. 週次サイクル

```
月曜: collectors/ 実行
火〜木: evaluators/ で評価
金曜: experiments/ で実験
土日: 週次サマリ作成
月曜: exports/ に出力
```

## 公開/非公開ポリシー

### 公開するもの（このリポジトリ）

- 判断思想
- 評価基準
- 判断フロー
- 週次サマリ
- 失敗パターン（一般化）

### 公開しないもの（.private/）

- 実データ
- 数値ログ
- 本番設定
- 運用閾値の実値

## AI 向け指示

### 収集・評価時

1. **Layer 1 は無視**: SNS ノイズ、表面的レビューは記録しない
2. **変化の種類を分類**: capability / constraint / pricing / other
3. **スコアリング基準に従う**: evaluators/README.md 参照

### 実験時

1. **時間を守る**: 90分超えたら強制終了
2. **本体に入れない**: experiments/ 内で完結
3. **必ず記録**: 結果を README.md に残す

### 出力時

1. **フォーマット遵守**: docs/weekly/ と exports/ の形式に従う
2. **数値は入れない**: 公開部分に実データを入れない

## 価値観（判断の拠り所）

1. **安全 > 速さ**: 迷ったら安全な方
2. **小さく試す > 大きく賭ける**: 確信がないならスモールスタート
3. **聞く > 推測**: 不確実なら人間に聞く
4. **記録 > 記憶**: 重要なことはファイルに残す
5. **自動化 > 手動**: 繰り返すなら自動化
6. **削除 > 放置**: 使わないものは消す
