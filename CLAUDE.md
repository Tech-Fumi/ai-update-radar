# AI Update Radar 開発ルール

## プロジェクト概要

AIの本質的な更新（能力・制限・価格）を検知・評価・実験するための運用リポジトリ。

**立ち位置**:
- ❌ 新しいAIツール集 / ニュースまとめ
- ✅ AI進化を"使える変化"に変換するフィルター

---

## 構成

```
ai-update-radar/
├── sources/           # 監視対象定義
├── collectors/        # 収集スクリプト
├── evaluators/        # 影響判定ロジック
├── experiments/       # PoC置き場（隔離）
├── docs/weekly/       # 週次サマリ
├── exports/           # 他リポジトリへの成果物
├── marketing/         # マーケティング機能
└── .private/          # 非公開データ（.gitignore対象）
```

---

## 運用ルール

### 情報の3層分類

| Layer | 対象 | アクション |
|:-----:|------|-----------|
| 1 | 90%のノイズ | 無視（見ない） |
| 2 | 変化検知のみ | 記録・週次サマリ |
| 3 | 深掘り対象 | 30-90分で実験 |

### 検知すべき変化（Layer 2 以上）

- **能力変化**: 新モデル、新機能、Agent拡張
- **制限解除**: コンテキスト長、レート制限緩和
- **価格変化**: 料金改定、新プラン

### 実験ルール（Layer 3）

- **時間上限**: 30〜90分
- **隔離環境**: experiments/ 内のみ
- **記録必須**: README.md で結果を記録

### 週次サイクル

```
月曜: collectors/ 実行
火〜木: evaluators/ で評価
金曜: experiments/ で実験
土日: 週次サマリ作成
月曜: exports/ に出力
```
