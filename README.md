# AI Update Radar

**AIの本質的な更新を検知・評価・実験するための運用リポジトリ**

> 全部追う人ほど疲弊し、作る人ほど勝つ。
> このリポジトリは「使える進化だけが自然に流れ込む構造」を実現する。

---

## 思想（Philosophy）

### なぜこのリポジトリが必要か

AI界隈の情報量は人間の処理限界を超えている：
- 新モデル（毎週〜毎日）
- 新ツール（Agents / Browser / IDE / MCP / RPA）
- 新フレームワーク
- 新API・新料金体系

**全部追うと**：情報疲労 → 判断力低下 → 実装が止まる → 収益化が遅れる

### このリポジトリの立ち位置

```
❌ 新しいAIツール集
❌ ニュースまとめ
❌ 技術デモ

✅ AI進化を"使える変化"に変換するフィルター
✅ 全プロジェクト共通の判断レイヤー
```

## 判断基準（Evaluation Criteria）

キャッチアップ対象を3層に分離する：

### Layer 1: 無視（90%）

- 「◯◯がすごい！」系SNS
- 表面的な比較レビュー
- UIが変わっただけのツール
- "〇〇をAIでやってみた"系

**→ 見ない・読まない・議論しない**

### Layer 2: 検知のみ（自動）

以下の変化だけを検知する：

| カテゴリ | 例 |
|----------|-----|
| **能力変化** | Browser常駐、Memory永続化、Agent間通信、OS操作、IDE常駐 |
| **制限解除** | コンテキスト長拡張、レート制限緩和、新機能アンロック |
| **価格変化** | 料金改定、新プラン、無料枠変更 |

**→ 詳細は読まない。「変化が起きたか」だけ検知**

### Layer 3: 深掘り（超少数）

以下の条件を**すべて**満たす場合のみ実験：

1. 今の自分の仕組みに
2. そのまま差し込める
3. 時間 or コスト or 人力を削減する

**→ 30〜90分上限で実験。ダメなら捨てて記録だけ残す**

## 公開/非公開の境界線（Transparency Policy）

### なぜ公開するのか

1. **信用の構築**: 思想・設計を先に言語化した人が標準になる
2. **説明コスト削減**: README が営業資料になる
3. **エコシステム形成**: 質問・改善案・実装者が集まる

### 公開するもの

| 項目 | 状態 |
|------|:----:|
| 判断思想 | ✅ 公開 |
| 評価カテゴリ | ✅ 公開 |
| 判断フロー | ✅ 公開 |
| 週次要約フォーマット | ✅ 公開 |
| やらなかった決定 | ✅ 公開 |
| 失敗パターン（一般化） | ✅ 公開 |

### 公開しないもの

| 項目 | 状態 |
|------|:----:|
| 実データ | ❌ 非公開 |
| 数値ログ | ❌ 非公開 |
| 顧客/案件情報 | ❌ 非公開 |
| 本番設定 | ❌ 非公開 |
| 運用閾値の実値 | ❌ 非公開 |

### 公開しても競合に負けない理由

```
設計はコピーできる。
運用の履歴はコピーできない。
信頼は履歴でしか作れない。
```

- 判断ログの蓄積
- 捨てた理由の記録
- 現場で調整された閾値
- 実運用で壊れた履歴
- 人間の介入判断の癖

**→ これらは「運用し続けた側」にしか生まれない**

## ディレクトリ構造

```
ai-update-radar/
├── sources/           # 監視対象定義（公式ブログ、GitHub releases、価格ページ等）
├── collectors/        # 収集スクリプト
├── evaluators/        # 影響判定ロジック（能力変化/制限解除/価格変化）

├── exports/           # 他リポジトリへ渡す成果物
└── .private/          # 非公開データ（.gitignore対象）
```

## 運用フロー

### 週次サイクル

```
月曜: 監視対象から変化を検知
      ↓
火〜木: Layer 2 → Layer 3 の選別
      ↓
金曜: 実験（30-90分上限）
      ↓
週末: 週次サマリ作成 → exports/
```

### 判断ログの義務化

すべての判断に以下を記録：

```markdown
## [YYYY-MM-DD] {対象名}

### 検知内容
- カテゴリ: 能力変化 / 制限解除 / 価格変化
- 概要:

### 判断
- 結果: 深掘り / 様子見 / 無視
- 理由:

### 実験結果（深掘りの場合）
- 所要時間:
- 結論: 採用 / 不採用
- 不採用理由:
```

## 価値観（Values）

すべての判断の拠り所：

| 優先 | 価値観 | 例 |
|:----:|--------|-----|
| 1 | **安全 > 速さ** | 迷ったら安全な方を選ぶ |
| 2 | **小さく試す > 大きく賭ける** | 確信がないならスモールスタート |
| 3 | **聞く > 推測** | 不確実なら人間に聞く |
| 4 | **記録 > 記憶** | 重要なことはファイルに残す |
| 5 | **自動化 > 手動** | 繰り返すなら自動化 |
| 6 | **削除 > 放置** | 使わないものは消す |
