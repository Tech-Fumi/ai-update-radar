# 設計思想

AI Update Radar のアーキテクチャと設計思想。

## なぜこのリポジトリが必要か

AI の更新情報は膨大。90% はノイズ。
このリポジトリは「使える変化」だけを抽出するフィルター。

## コア思想

### 1. 3層分類

```
全情報 → Layer 1（無視）
       → Layer 2（記録）
       → Layer 3（実験）
```

### 2. 隔離実験

- 実験は `experiments/` 内のみ
- 本番コードに影響しない
- 時間上限 30-90分

### 3. 成果物の分離

- 判断過程: このリポジトリ
- 採用結果: `exports/` → 他リポジトリ

## 他リポジトリとの関係

```
ai-update-radar
    ↓ exports/
    ├── infra-automation（MCP 統合）
    ├── ScrimAutomationEngine
    └── StreamFlowEngine
```

## 公開/非公開の分離

| 公開 | 非公開 |
|------|--------|
| 判断思想 | 実データ |
| 評価基準 | 運用閾値 |
| 週次サマリ | 数値ログ |
