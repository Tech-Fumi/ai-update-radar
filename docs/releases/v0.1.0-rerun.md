# v0.1.0-rerun

## Added
- Retry button (failed runs only) to re-submit the same payload.
- Re-run button (always available) to re-execute any run.
- New API: `GET /runs/{run_id}/task` returns the original task payload for replay.
- Viewer automatically navigates to a filtered Runs list by the new `trace_id`.

## Traceability
- `trace_id` format: `tr_rerun_<old_run_id>_<rand>`
- Parent/child relationship is trackable via `trace_id` (and logs).
